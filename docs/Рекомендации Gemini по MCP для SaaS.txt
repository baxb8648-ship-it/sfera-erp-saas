Для развития модульной ERP-CRM платформы и перехода к B2B SaaS-экосистеме идеально подойдут следующие MCP-серверы:

**1. Pinecone**

* **Зачем нужен:** В дорожной карте (Вариант А) приоритетным шагом указано внедрение RAG-системы.


* **Применение:** Сервер позволит интегрировать базу знаний (прошлые коммерческие предложения, ГОСТы, СНиПы, прайс-листы) в векторную базу данных для генерации ответов ИИ в контексте специфики работ по АКЗ и огнезащите.



**2. Prisma**

* **Зачем нужен:** Архитектурное исследование требует перехода с SQLite на PostgreSQL для реализации Multi-tenancy (многоарендности) и поддержки динамических JSONB полей.


* **Применение:** Prisma позволит ИИ-агенту управлять базами данных PostgreSQL, автоматически создавать миграции и генерировать типобезопасные запросы.



**3. Chrome DevTools for agents**

* **Зачем нужен:** В проекте активно работают парсеры коммерческих площадок (Фабрикант, Росэлторг, ТЭК-Торг) и механизм снайпер-кампаний для сбора B2B-лидов.


* **Применение:** Агент сможет напрямую управлять браузером Chrome для глубокого веб-скрейпинга динамических сайтов. Это поможет обходить сложные блокировки провайдеров и защиты вроде DDoS-Guard/StormWall, с которыми система сталкивалась ранее.



**4. Sequential Thinking**

* **Зачем нужен:** Проект передает локальной модели Ollama (Qwen2:7B) сложные аналитические задачи, такие как скоринг лидов, прогнозирование конкуренции и анализ рисков в договорах заказчиков.


* **Применение:** Этот сервер наделит языковую модель способностью пошагового структурированного рассуждения, что критически повысит точность алгоритмических и логических выводов ИИ.



**5. Postman**

* **Зачем нужен:** Система завязана на множестве внешних интеграций: api-fns.ru для ЕГРЮЛ, Groq Whisper API для голосовых задач и Telegram Bot API.


* **Применение:** Агент получит прямой доступ к тестированию API-эндпоинтов, анализу их структуры и автоматической генерации автотестов для проверки лимитов и работоспособности интеграций.



**6. Neon или AlloyDB for PostgreSQL**

* **Зачем нужен:** Переход от внутренней CRM к публичному SaaS-продукту потребует надежной облачной инфраструктуры баз данных для изоляции данных клиентов (tenant_id).


* **Применение:** Эти серверы дадут ИИ возможность взаимодействовать с облачным PostgreSQL на естественном языке, делать выборки и управлять структурой таблиц.



**7. GitHub или GitLab Orbit**

* **Зачем нужен:** Кодовая база содержит множество микросервисов (бэкенд FastAPI, фронтенд React/Vite, скрипты туннелирования start_tunnel.py).


* **Применение:** Агент сможет проводить аудит кода, помогать с деплоем на VPS и управлять коммитами напрямую из среды разработки.
