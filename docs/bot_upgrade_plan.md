## 📈 Вариант 1: Прокачать существующего бота

### 🔴 Высокий приоритет (быстро, высокая ценность)

#### 1. Команда `/crm_summary` — Дашборд одной кнопкой
Пользователь пишет `/дашборд` или `/status` и получает мгновенную сводку:
```
📊 ЛЕОНИКА CRM — сводка на 30.06.2026
━━━━━━━━━━━━━━━━━━━━
📁 Задачи:     12 активных, 3 новых, 2 просрочены
🏗 Объекты:    5 в работе, 2 на паузе  
🎯 Тендеры:    8 новых, 1 дедлайн завтра
💰 Финансы:    +320 000 ₽ за июнь
👥 Лиды:       47 в базе
━━━━━━━━━━━━━━━━━━━━
```
**Реализация:** эндпоинт `/dashboard` уже есть. Нужно просто добавить `/status` команду в `telegram_webhook.py`.

---

#### 2. Команда `/мои_задачи` — персональная Kanban в боте
```
📌 Ваши задачи (@иванов):
━━━━━━━━━━━━━━━━━━━━
🔴 Срочно: АКЗ резервуар СОФ-1 [сегодня]
🟡 В работе: Замер на ОНПЗ [завтра]
🟢 Новая: Подготовить КП Газпром  
━━━━━━━━━━━━━━━━━━━━
[+ Добавить задачу]  [Открыть CRM]
```
**Реализация:** нужна привязка Telegram ID → CRM User (поле `telegram_chat_id` уже есть в модели User!).

---

#### 3. Быстрые команды управления задачами
- `/задача [текст]` — создать задачу текстом (без голоса)
- `/завершить [ID]` — закрыть задачу
- `/перенести [ID] [дата]` — изменить срок

---

#### 4. Фото-отчёт со стройки → автоприкрепление к объекту
Сотрудник фотографирует объект и отправляет боту.  
Бот спрашивает: *"К какому объекту прикрепить?"* → выпадающий список.  
Фото сохраняется в CRM в карточке объекта.

**Реализация:** добавить обработчик `message.photo` в `telegram_webhook.py`, вызов `/objects` API.

---

#### 5. Умный ИИ-Копилот v2 с контекстом CRM

Сейчас `@ai` знает только системный промпт про АКЗ.  
Апгрейд: перед вызовом Ollama бот запрашивает свежие данные из БД:
```python
# Динамический контекст
context = f"""
Текущие задачи: {get_active_tasks_summary()}
Активные тендеры: {get_active_tenders_summary()}
Склад ЛКМ: {get_inventory_summary()}
"""
```
Теперь менеджер может спросить: *"@ai сколько Унипол-3030 на складе?"* и получить реальный ответ.

---

### 🟡 Средний приоритет (ценная автоматизация)

#### 6. Авто-КП по запросу в Telegram
Менеджер пишет:  
*"@ai сделай КП для Газпрома, АКЗ 2000 м², Унипол-3030, срок 45 дней"*

Бот → Ollama генерирует КП → PDF создается через существующий `document_generator.py` → PDF отправляется прямо в чат.

#### 8. Кнопка "Напомнить через N дней" под тендером
Inline-кнопка `⏰ Напомнить` → бот уточняет через сколько дней → сохраняет в БД → scheduler отправляет личное напоминание.

---

#### 9. Обратная связь по задачам — статус-апдейты
Сотрудник в поле меняет статус через бота:  
`/статус 42 Выполнена` → задача #42 в CRM закрыта + уведомление руководителю.

## 🤝 Вариант 2: Создать бота-Коллегу (отдельный специализированный бот)

Идея: второй бот с **узкой специализацией**, работающий рядом с основным.

### 🎯 Концепция: **"Снабженец"** — бот управления складом и материалами

Основной бот — универсальный CRM-менеджер.  
Снабженец — специалист по ЛКМ, складу, закупкам.

#### Функции "Снабженца":
| Команда | Действие |
|---|---|
| `/остатки` | Текущий склад ЛКМ с критическими позициями 🔴 |
| `/списать [материал] [кол-во] [объект]` | Списание со склада через бота |
| `/заказать [материал] [кол-во]` | Создает задачу закупки |
| `/расход [объект]` | Статистика расхода ЛКМ по объекту |
| `/норма [площадь] [материал]` | Расчёт нормы расхода ЛКМ через ИИ |
| `/фото [объект]` | Загрузка фото на объект |

**Почему отдельный бот?** Разные аудитории: прорабы/снабженцы используют только склад — им не нужен весь CRM. Боты можно добавить в разные группы.
